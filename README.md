# CRM-Style Admin Panel Automation

## 프로젝트 배경
더존비즈온 CRM QA 담당 시절, 요금제/프로모션 기능 검수를 수동으로 
반복 수행하며 회귀 테스트 자동화의 필요성을 느꼈습니다. 
사내 서비스는 보안상 공개할 수 없어, 구조적으로 유사한 
OrangeHRM 데모를 대상으로 동일한 설계 원칙을 적용했습니다.

## 왜 이 대상을 선택했는가
- CRUD, 필터/검색, 권한별 화면 노출 등 CRM 관리화면과 
  동일한 패턴을 가진 공개 데모 사이트
- 로그인 → 리스트 조회 → 등록/수정/삭제 흐름이 실무 시나리오와 
  구조적으로 일치

## 아키텍처
[POM 다이어그램 또는 폴더 구조 설명]
왜 Page Object Model을 택했는가: 
UI 변경 시 로케이터 수정 지점을 한 곳으로 집중시켜 
유지보수 비용을 낮추기 위함

## 테스트 전략
- 12개 시나리오: 기능(6) / 데이터 정합성(2) / 경계값(2) / API(1) / 크로스브라우저(1)
- 데이터 기반 테스트로 케이스 확장 시 코드 수정 없이 JSON만 추가

## CI/CD
- Push 시 자동 실행 + 매일 07시(KST) 정기 실행
- 실패 시 스크린샷/트레이스 자동 첨부

## 트러블슈팅 (가장 중요한 섹션)
- 문제: 등록 후 리스트 반영 확인 시 flaky 발생
- 원인: 비동기 렌더링 완료 전에 assertion 실행
- 해결: waitForResponse로 API 응답 완료 시점을 명시적으로 대기하도록 변경

## 향후 개선 방향
- Visual regression 테스트 추가
- Allure 리포트 연동으로 실패 트렌드 시각화
